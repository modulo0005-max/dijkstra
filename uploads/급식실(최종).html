📐 지도교사의 수학 노트: 간선 완화(Relaxation)와 대기 효율

직관적으로는 음료수대를 거쳐가는 넓은 길인 S에서 A로 바로 가는 경로(6초)가 가장 좋아 보입니다. 그러나 미끄러운 바닥(B, 3초)을 조심히 거쳐가며 A로 우회하여 돌아가는 길을 탐색하면 3 + 2 = 5초 만에 갈 수 있죠! 기존에 가늠했던 A의 최단 시간을 6초에서 5초로 '완화(Relaxation)'시키는 과정입니다. 이처럼 최단 동선은 우회로를 통해 새롭게 완성되곤 합니다. (S ➡️ B ➡️ A ➡️ T = 3 + 2 + 3 = 8초)

📐 지도교사의 수학 노트: 최적 부분 구조(Optimal Substructure)

다익스트라 알고리즘은 '부분의 최적 경로들이 모여 전체 최적 경로가 된다'는 동적 계획법(Dynamic Programming)의 성질을 따릅니다. 오늘의 국 배식처(D)에서 치즈 닭갈비 배식처(M)로 직행하면 9초가 걸리지만, 샐러드/김치(F)를 거쳐 가면 2 + 3 = 5초 만에 갈 수 있죠. 이렇듯 각 세부 목적지까지 도달하는 부분적인 최적 시간들을 신뢰할 수 있게 축적함으로써 최종 목적지(M)까지의 완벽한 12초 경로를 찾아냅니다. (T ➡️ C ➡️ D ➡️ F ➡️ M = 4 + 3 + 2 + 3 = 12초)

📐 지도교사의 수학 노트: 탐욕적 선택 조건(Greedy Choice Property)

다익스트라 알고리즘은 매 순간 **'가장 비용이 작게 누적되어 고정된 확실한 노드'**만을 전제하여 탐색 영역을 넓혀나가는 '탐욕적 알고리즘(Greedy Algorithm)'의 모범입니다. M에서 탐색을 시작할 때, 아직 방문 안 한 노드 중 가중치가 3으로 가장 최소인 가운데 줄(I)을 우선 순위에 올립니다. 그리고 I에서 연결된 시야를 이용해 창가석(H)까지의 비용을 기존 6에서 5(3+2)로 완화하죠. 이 탐욕적이고 엄밀한 순차 증명을 통해, 인간의 단순한 직관적 오차를 제압하고 항상 정답인 '11초 최단 경로'를 산출해 낼 수 있습니다. (M ➡️ I ➡️ H ➡️ J ➡️ G = 3 + 2 + 3 + 3 = 11초)

📐 0. 다익스트라 알고리즘의 핵심 수학적 원리

코드로만 구현되는 줄 알았던 다익스트라 알고리즘의 이면에는 사실 매우 정교하고 정밀한 수학적 성질들과 논증 구조가 깔려있습니다. 탐구보고서(세특)를 작성할 때 아래의 증명을 꼭 인용해 보세요!

① 그래프 이론을 활용한 수학적 모델링

우선 해결하려는 문제를 수학적 구조인 유향 가중치 그래프(Directed Weighted Graph) \(G = (V, E)\)로 추상화합니다.
여기서 \(V\)는 점들의 집합(급식 코너), \(E\)는 노드를 잇는 실선 집합(복도 및 이동 통로)입니다.
각 간선 \(e \in E\)에 대하여 양의 실수 가중치를 부여하는 함수를 \(w : E \rightarrow \mathbb{R}^+\)로 정의합니다.
시작점 \(S \in V\)로부터 임의의 정점 \(v \in V\)에 이르는 경로를 \(P = (v_0, v_1, \dots, v_k)\) (단, \(v_0 = S\), \(v_k = v\))라 할 때, 경로의 총비용 \(w(P)\)는 다음과 같은 수열의 합으로 나타낼 수 있습니다. \[ w(P) = \sum_{i=1}^{k} w(v_{i-1}, v_i) \] 최단 경로 탐색 문제는 바로 이 \(w(P)\)를 최소화하는 함수값을 찾아내는 과정입니다.

② 삼각 부등식(Triangle Inequality) 조건

임의의 세 정점 \(u, v, y \in V\) 사이의 실제 최단 거리를 \(\delta(u, v)\)라고 정의할 때, 언제나 기하학적 정점 관계 상에서 다음의 삼각 부등식이 성립합니다. \[ \delta(u, v) \le \delta(u, y) + \delta(y, v) \] 이 부등식은 지름길 탐색 논리의 대전제가 됩니다. 즉, 우회하는 비용(\(\delta(u, y) + \delta(y, v)\))이 직접 통과하는 실제 비용(\(\delta(u, v)\))보다 절대로 작을 수는 없다는 공리입니다. 다익스트라는 이 조건 하에서 정점들의 탐색 가능 범위를 안정적으로 수축해 갑니다.

③ 간선 완화(Edge Relaxation)의 부등식 정식화

알고리즘이 동작하는 동안, 시작점 \(S\)로부터 정점 \(v\)까지의 누적 비용의 추정값을 \(d[v]\)라 합시다.
최초 상태에서는 \(d[S] = 0\), 그 외의 모든 \(v \in V \setminus \{S\}\)에 대해 \(d[v] = \infty\)로 초기 설정합니다.
알고리즘의 매 회차마다 임의의 연결된 두 정점 \(u, v\)에 대하여 다음과 같은 최적화 연산(완화)을 반복적으로 적용합니다. \[ d[v] = \min \big( d[v],\, d[u] + w(u, v) \big) \] 수학적으로 만약 \(d[v] > d[u] + w(u, v)\)인 관계 부등식이 성립한다면, 더 빠른 우회로를 찾은 것이므로 \(d[v]\)를 \(d[u] + w(u, v)\)로 재정의(대입)합니다. 이 부등식 변환 과정을 통해 추정값 \(d[v]\)는 항상 실제 최단 거리인 \(\delta(S, v)\)로 단조 감소하며 수렴하게 됩니다.

④ 수학적 귀납법(Mathematical Induction)을 통한 알고리즘 정당성 증명

다익스트라 알고리즘이 임의의 정점을 방문 처리 집합 \(S\)에 추가할 때, "그 시점에서의 거리값 \(d[u]\)가 항상 실제 최단 거리 \(\delta(S, u)\)와 완벽하게 일치한다"는 성질을 수학적 귀납법으로 엄밀하게 증명할 수 있습니다.

[1] 초기 단계 (Base Case):
집합 \(S = \emptyset\) 일 때, 시작점 \(S\)를 집합에 넣습니다. 이때 \(d[S] = 0 = \delta(S, S)\)이므로 자명하게 성립합니다.

[2] 가정 및 귀납 단계 (Inductive Step):
이미 \(k\)개의 정점이 포함되어 \(d[v] = \delta(S, v)\)가 모두 만족하는 집합 \(S\)가 있다고 가정합시다. 이제 집합 \(S\) 밖에서 \(d[u]\)가 최소인 정점 \(u\)를 새로 집합에 추가하려 합니다.
이때 만약 \(d[u] \neq \delta(S, u)\)라고 가정해 봅시다. (귀류법 적용)
실제 최단 경로 상에서 집합 \(S\)를 벗어나기 직전의 노드를 \(x\), 벗어난 직후의 노드를 \(y\)라 할 때, 최단 경로 성질에 의해 다음 부등식 관계가 도출됩니다. \[ d[y] = \delta(S, y) \le \delta(S, u) < d[u] \] 그러나 알고리즘은 매회 탐욕적 선택 조건에 의해 집합 \(S\) 밖의 후보군 중 가장 최소값을 가지는 정점을 고릅니다. 따라서 \(d[u] \le d[y]\)이어야만 합니다. 이는 앞서 도출한 \(d[y] < d[u]\)와 정면으로 모순(Contradiction)됩니다.

따라서 새로 선택한 정점 역시 \(d[u] = \delta(S, u)\)일 수밖에 없습니다. 수학적 귀납법에 의해 다익스트라 알고리즘이 산출하는 모든 정점의 최단 거리 추정값은 실제 이론상 최단 경로와 100% 일치함이 완벽히 증명됩니다.
